# Техническое задание: Volume Optimizer PWA

## Приложение для автоматического подбора и корректировки тренировочного объёма

**Версия документа:** 1.0
**Дата:** 28.03.2026

## 1. Общее описание продукта

### 1.1 Цель

PWA-приложение, которое на основе данных о выполненных тренировках автоматически подбирает оптимальный тренировочный объём для каждого упражнения, управляет периодизацией и адаптирует нагрузку под индивидуальный отклик атлета.

### 1.2 Целевая аудитория

Натуральные атлеты среднего уровня (1-5 лет опыта), тренирующиеся 2-4 раза в неделю, которые хотят систематически прогрессировать и не могут или не хотят подбирать объём интуитивно.

### 1.3 Ключевой сценарий

Атлет весом 117 кг, рост 198 см, подтягивается 2-3 раза (10 раз с резиной). Тренируется 2 раза в неделю. Прогресс встал. Приложение должно:
- определить стартовый объём (сколько подходов/повторений)
- решить, когда работать с резиной, когда без
- каждую неделю корректировать нагрузку на основе фактических данных
- автоматически встраивать разгрузочные недели
- давать чёткие рекомендации: «сегодня делай 4×6 с резиной, RPE 7-8»

### 1.4 Поддерживаемые типы упражнений

| Тип | Варианты нагружения | Пример |
|-----|---------------------|--------|
| Собственный вес | Без помощи / с резиной (ассистент) | Подтягивания, отжимания на брусьях |
| Гантели | Дискретные веса (задаются пользователем) | Жим гантелей, тяга в наклоне, сгибания |

## 2. Научная основа

### 2.1 Система Volume Landmarks (Renaissance Periodization, Israetel)

Приложение оперирует четырьмя ключевыми объёмными ориентирами для каждой мышечной группы:

| Ориентир | Определение | Типичный диапазон (подходов/неделю) |
|----------|-------------|-------------------------------------|
| **MV** (Maintenance Volume) | Объём для удержания текущей массы | ~6 |
| **MEV** (Minimum Effective Volume) | Минимальный объём, стимулирующий рост | 8-10 |
| **MAV** (Maximum Adaptive Volume) | Оптимальная зона максимального прогресса | 12-20 |
| **MRV** (Maximum Recoverable Volume) | Потолок, за которым восстановление невозможно | 20-25 |

Эти значения — не фиксированные числа, а индивидуальные ориентиры, которые приложение уточняет на основе данных пользователя.

### 2.2 Гибридная модель нагружения: % от максимума + RPE-корректор

Система использует **процентный подход** (по принципам СРЦ Головинского) как основу, с RPE как корректором текущего состояния.

#### 2.2.1 Процентная основа (СРЦ)

Для каждого упражнения хранится **репетиционный максимум** (repMax) — максимальное количество повторений в заданном режиме. Рабочие повторения задаются как % от repMax:

| Зона нагрузки | % от repMax | Назначение |
|---------------|-------------|------------|
| Лёгкая | 40-55% | Разминка, деload, техника |
| Средняя | 55-70% | Объёмная работа, накопление тоннажа |
| Тяжёлая | 70-85% | Силовая работа, максимальная адаптация |
| Пиковая | 85-95% | Реализация, тесты прогресса |

Пример: repMax (подтягивания с резиной) = 10. Тяжёлая сессия: 75% = 7-8 повторений.

#### 2.2.2 RPE как корректор (не основа)

RPE используется **после подхода** для корректировки плана, а не для задания нагрузки:

| RPE | RIR | Описание | Корректировка |
|-----|-----|----------|---------------|
| 10 | 0 | Отказ | Следующий подход: -1 повтор или -вес |
| 9 | 1 | Почти отказ | Оставить как есть |
| 8 | 2 | Тяжело, но контроль | Целевая зона — идеально |
| 7 | 3 | Умеренно | Можно прибавить нагрузку |
| 6 | 4+ | Легко | Прибавить вес или повторения |

Целевой RPE рабочих подходов: **7-9**. Если факт. RPE отклоняется от целевого — система корректирует следующий подход в реальном времени.

#### 2.2.3 Коэффициент прогрессии (из СРЦ)

Каждую неделю repMax пересчитывается:

```
Новый repMax = Текущий repMax × (1 + progressionCoefficient)

progressionCoefficient (дефолт по уровню):
  absolute_beginner: 1.5% / неделю
  beginner:          1.0% / неделю
  intermediate_early: 0.75% / неделю
  intermediate_late:  0.5% / неделю
  advanced:           0.25% / неделю
```

Коэффициент корректируется авторегуляцией (раздел 3.5): при хорошем прогрессе +0.25%, при плохом -0.25%, при необходимости деload → 0%.

### 2.3 Модель периодизации: развязка объёма и интенсивности (принцип СРЦ)

В отличие от западной модели (где объём и интенсивность растут одновременно), приложение использует **развязанную волновую периодизацию** по принципу СРЦ Головинского: объём и интенсивность чередуются, а не нарастают параллельно.

```
Мезоцикл (4-6 недель + деload):

Неделя 1 (накопление):
  Объём: MEV          Интенсивность: средняя (55-70% repMax)
  Цель: адаптация, набор базового тоннажа

Неделя 2 (интенсификация):
  Объём: MEV          Интенсивность: тяжёлая (70-85% repMax)
  Цель: силовой стимул при том же объёме

Неделя 3 (накопление+):
  Объём: MEV + 2 подх  Интенсивность: средняя (55-70% repMax)
  Цель: рост объёма на фоне сниженной интенсивности

Неделя 4 (интенсификация+):
  Объём: MEV + 2 подх  Интенсивность: тяжёлая (70-85% repMax)
  Цель: силовой стимул при повышенном объёме

Неделя 5 (опционально, пик):
  Объём: MAV           Интенсивность: тяжёлая (75-85% repMax)
  Цель: пиковая нагрузка перед деload

Деload:
  Объём: MV            Интенсивность: лёгкая (40-55% repMax)
  Цель: восстановление, суперкомпенсация
```

**Волновое микроциклирование внутри недели** (2 тренировки):

| Сессия | Тип | Объём | Интенсивность | Назначение |
|--------|-----|-------|---------------|------------|
| 1 | Тяжёлая | 45% недельного | 70-85% repMax | Силовой стимул |
| 2 | Лёгкая/средняя | 55% недельного | 55-70% repMax | Объём, тоннаж, техника |

Это позволяет внутри одной недели давать и силовой стимул, и объёмную работу, не перегружая восстановление.

### 2.4 Специфика подтягиваний для тяжёлых атлетов

На основе исследований Pavel Tsatsouline (StrongFirst), Fighter Pull-up Program, метода Grease the Groove, а также программ подтягиваний из СРЦ Головинского:

**Принципы:**
- Для атлета 100+ кг подтягивание — это «тяжёлое» упражнение даже с собственным весом
- Резина снижает нагрузку, позволяя набрать объём и тоннаж
- Прогрессия: больше повторений с резиной → меньше помощи резины → собственный вес
- Эксцентрическая фаза (медленное опускание 4-6 сек) — мощный стимул, так как мышцы на ~150% сильнее в эксцентрике
- При 2 тренировках в неделю: **тяжёлая** (собственный вес / эксцентрика, меньше повторений) + **лёгкая/средняя** (резина, больше повторений, набор тоннажа) — по принципу волнового микроциклирования СРЦ
- repMax пересчитывается еженедельно с коэффициентом прогрессии; переход между фазами (резина → BW) происходит автоматически по достижении пороговых repMax

## 3. Алгоритмическая логика

### 3.1 Инициализация пользователя

При первом запуске собираются данные:

```typescript
interface UserProfile {
  bodyweight: number;        // кг
  height: number;            // см
  age: number;
  trainingAgeMonths: number; // стаж в месяцах (точнее чем категории)
  sessionsPerWeek: number;   // 2-4
  goals: 'strength' | 'hypertrophy' | 'both';
}

// Категория опыта вычисляется из trainingAgeMonths:
type TrainingLevel = 'absolute_beginner'  // 0-6 мес: нейромышечная адаптация
                   | 'beginner'           // 6-18 мес: быстрый линейный прогресс
                   | 'intermediate_early' // 18-36 мес: прогресс замедляется, нужна периодизация
                   | 'intermediate_late'  // 36-60 мес: медленный прогресс, тонкая настройка объёма
                   | 'advanced';          // 60+ мес: минимальный прогресс, сложная периодизация

function getTrainingLevel(months: number): TrainingLevel {
  if (months < 6) return 'absolute_beginner';
  if (months < 18) return 'beginner';
  if (months < 36) return 'intermediate_early';
  if (months < 60) return 'intermediate_late';
  return 'advanced';
}
```

```typescript
interface ExerciseConfig {
```

#### 3.2.1 Начальные Volume Landmarks (дефолты)

Определяются по `trainingLevel` (вычисляется из `trainingAgeMonths`):

```
absolute_beginner (0-6 мес):
  mv=3, mev=4, mav=8, mrv=12
  // Нейромышечная адаптация: растёт от минимального объёма

beginner (6-18 мес):
  mv=4, mev=6, mav=12, mrv=16
  // Линейный прогресс: быстро отвечает на нагрузку

  mv=5, mev=8, mav=14, mrv=20
  // Замедление: нужна периодизация и авторегуляция

intermediate_late (36-60 мес):
  mv=6, mev=10, mav=16, mrv=22
  // Медленный прогресс: тонкая настройка объёма критична

advanced (60+ мес):
  mv=8, mev=12, mav=20, mrv=26
```

Эти значения пользователь может скорректировать вручную.
Напоминание: для bodyweight упражнений landmarks дополнительно корректируются по весу атлета (см. раздел 8.9).

### 3.3 Структура мезоцикла

```typescript
interface Mesocycle {
  id: string;
  exerciseId: string;
  startDate: Date;

  // Конфигурация
  durationWeeks: number;       // 4-6 (определяется автоматически)
  deloadWeek: boolean;         // всегда true

  // Недельные планы
  weeks: WeekPlan[];

  // Состояние
  status: 'active' | 'completed' | 'deload';
}

interface WeekPlan {
  weekNumber: number;
  weekType: 'accumulation' | 'intensification' | 'peak' | 'deload';
  targetSets: number;          // общее кол-во подходов за неделю
  intensityZone: 'light' | 'medium' | 'heavy' | 'peak'; // % от repMax
  targetRPE: number;           // целевой RPE (7-9) — корректор, не основа

  sessions: SessionPlan[];     // распределение по тренировкам
}

interface SessionPlan {
  sessionNumber: number;       // 1 или 2 (при 2х в неделю)
  sets: number;
  repsTarget: number;          // вычисляется как % от repMax
  intensityPercent: number;    // % от repMax (40-95)
  targetRPE: number;           // целевой RPE — для валидации
  loadConfig: LoadConfig;
  sessionType: 'heavy' | 'light'; // тяжёлая / лёгкая (волновое микроциклирование СРЦ)
}
```

#### 3.3.1 Алгоритм генерации мезоцикла (модель СРЦ + Volume Landmarks)

```
ВХОД: exerciseConfig, userProfile, previousMesocycleData (если есть)

1. Определить длину мезоцикла:
   - absolute_beginner (0-6 мес) → 6 недель + 1 деload = 7 недель
   - beginner (6-18 мес) → 5 недель + 1 деload = 6 недель
   - intermediate_early (18-36 мес) → 4 недели + 1 деload = 5 недель
   - intermediate_late (36-60 мес) → 4 недели + 1 деload = 5 недель
   - advanced (60+ мес) → 3 недели + 1 деload = 4 недели

2. Определить стартовый объём и repMax:
   - Если первый мезоцикл → объём = MEV, repMax = тестовый максимум
   - Если повторный → объём = MEV + 1-2 подхода (не более MAV)
     repMax = последний пересчитанный repMax

3. Расставить типы недель (РАЗВЯЗКА объёма и интенсивности по СРЦ):
   Чередование: накопление → интенсификация → накопление+ → интенсификация+ → [пик] → деload

   Пример для 5-недельного мезоцикла (intermediate):
     Неделя 1: accumulation   — объём MEV,        интенсивность medium (55-70%)
     Неделя 2: intensification — объём MEV,        интенсивность heavy (70-85%)
     Неделя 3: accumulation   — объём MEV+2 подх,  интенсивность medium (55-70%)
     Неделя 4: intensification — объём MEV+2 подх,  интенсивность heavy (70-85%)
     Неделя 5: deload         — объём MV,          интенсивность light (40-55%)

   Для 6-недельного (beginner):
     Неделя 1: accumulation   — MEV,       medium
     Неделя 2: accumulation   — MEV,       medium  (удвоенная адаптация)
     Неделя 3: intensification — MEV,       heavy
     Неделя 4: accumulation   — MEV+2,     medium
     Неделя 5: intensification — MEV+2,     heavy
     Неделя 6: deload         — MV,        light

4. Рассчитать повторения из % от repMax:
   repsTarget = round(repMax × intensityPercent)
   Где intensityPercent берётся из зоны нагрузки:
     medium: 60-65% repMax
     heavy:  75-80% repMax
     light:  45-50% repMax

5. Распределить подходы по сессиям (волновое микроциклирование СРЦ):
   Если sessionsPerWeek = 2:
     Сессия 1 (тяжёлая): 45% подходов, intensityPercent + 10%
     Сессия 2 (лёгкая):  55% подходов, intensityPercent - 10%

   Это даёт внутри каждой недели чередование:
     Тяжёлая сессия: меньше подходов, больше нагрузка (ближе к макс.)
     Лёгкая сессия: больше подходов, меньше нагрузка (тоннаж, техника)

6. Еженедельный пересчёт repMax (ключевой принцип СРЦ):
   repMax_новый = repMax_текущий × (1 + progressionCoefficient)
   Рабочие повторения автоматически растут вслед за repMax

7. Деload неделя (последняя):
   sets = MV (maintenance)
   reps = 45-50% от repMax
   intensityPercent = 40-55% (лёгкая зона)
   targetRPE = 5-6
```

### 3.4 Алгоритм выбора нагрузки (LoadConfig)

#### 3.4.1 Подтягивания (bodyweight + band)

```typescript
interface BodyweightLoadConfig {
  mode: 'bodyweight' | 'band_assisted' | 'eccentric_only';
  bandAssistPercent?: number;
  eccentricSeconds?: number;  // длительность негативной фазы
}
```

**Дерево решений:**

```
ВХОД: maxReps_bodyweight, maxReps_band, targetReps, targetRPE

1. Определить «эффективный максимум» для сессии:
   effectiveMax_bw = maxReps_bodyweight
   effectiveMax_band = maxReps_band

2. Если targetReps <= effectiveMax_bw * 0.7:
   → mode = 'bodyweight'
   (можем сделать целевые повторения с запасом RPE)

3. Если targetReps > effectiveMax_bw * 0.7 И targetReps <= effectiveMax_band * 0.7:
   → mode = 'band_assisted'

4. Если effectiveMax_bw < 3:
   → Объёмная сессия: mode = 'band_assisted'
   → Силовая сессия: mode = 'bodyweight' (1-2 повтора) + 'eccentric_only' (дополнительно)

5. Эксцентрическая добавка:
   Если eccentricOption = true И mode = 'bodyweight' И reps < 4:
   → добавить 2-3 эксцентрических повтора по 5 секунд после основных

ПРИМЕР для пользователя (BW 117кг, 2-3 reps BW, 10 reps band):
  Сессия 1 (объём): 4×8 с резиной, RPE 7-8
  Сессия 2 (сила): 3×2 BW + 2×3 эксцентрика 5с, RPE 8-9
```

#### 3.4.2 Гантели (дискретные веса)

```typescript
interface DumbbellLoadConfig {
  weight: number;             // кг — из массива availableWeights
  repRange: [number, number]; // например [6, 10]
}
```

**Алгоритм Double Progression:**

```
ВХОД: availableWeights[], currentWeight, currentMaxReps, targetRepRange[min, max]

1. Если currentMaxReps >= targetRepRange[max]:
   → Повышаем вес до следующего доступного
   → Сбрасываем целевые повторения к targetRepRange[min]

   nextWeight = availableWeights.find(w => w > currentWeight)

   // Проверка величины прыжка
   jumpPercent = (nextWeight - currentWeight) / currentWeight * 100

   Если jumpPercent > 20%:
     → Расширить repRange вниз: min = min - 2
     → Предупредить пользователя: «большой прыжок веса, начинаем с меньших повторений»

2. Если currentMaxReps >= targetRepRange[min] И < targetRepRange[max]:
   → Продолжаем с текущим весом, целим на +1-2 повтора

3. Если currentMaxReps < targetRepRange[min]:
   → Вес слишком тяжёлый, откат к предыдущему
   previousWeight = availableWeights.findLast(w => w < currentWeight)

ПРИМЕР:
  availableWeights = [5,7,9,11,13,15,18,20,22,25,27,29,32,34,36,38,40]
  currentWeight = 22, maxReps = 10, range = [6, 10]
  → Переход на 25 кг (прыжок ~14%, < 20% — нормально)
  → Range остаётся [6, 10]
  → Начинаем 25кг × 6-7 повторений

ПРИМЕР с большим прыжком:
  currentWeight = 29, maxReps = 10, range = [6, 10]
  → Переход на 32 кг (прыжок ~10%, < 20% — нормально)
  → Начинаем 32кг × 6-7 повторений
  Примечание: при этой раскладке гантелей прыжки ≤15%, расширение range не нужно
```

### 3.5 Алгоритм авторегуляции после каждой сессии

Это **ядро** приложения. После каждой тренировки пользователь вводит фактические данные.

```typescript
interface SessionLog {
  date: Date;
  exerciseId: string;
  sets: SetLog[];

  // Субъективные метрики (заполняет пользователь)

  pumpQuality: 1 | 2 | 3;    // 1=нет, 2=умеренный, 3=сильный
  sorenessLevel: 0 | 1 | 2 | 3; // от предыдущей тренировки. 0=нет, 3=сильная
  jointPain: boolean;         // боль в суставах (да/нет)
```

#### 3.5.1 Дерево решений корректировки объёма

После каждой сессии система анализирует данные и корректирует план следующей тренировки:

```
ВХОД: sessionLog, currentWeekPlan, mesocycle

// Шаг 1: Вычислить Performance Score
performanceScore = calculatePerformanceScore(sessionLog)

function calculatePerformanceScore(log):
  score = 0

  // Фактические повторения vs план
  actualTotalReps = sum(log.sets.map(s => s.reps))
  plannedTotalReps = plan.sets * plan.reps
  repRatio = actualTotalReps / plannedTotalReps

  if repRatio >= 1.05: score += 2    // перевыполнение
  if repRatio >= 0.95: score += 1    // в норме
  if repRatio < 0.90: score -= 1     // недовыполнение
  if repRatio < 0.80: score -= 2     // значительное падение

  // RPE отклонение
  avgRPE = average(log.sets.map(s => s.rpe))
  rpeDeviation = avgRPE - plan.targetRPE

  if rpeDeviation <= -1: score += 1   // легче чем планировалось — хорошо
  if rpeDeviation >= 1: score -= 1    // тяжелее чем планировалось
  if rpeDeviation >= 2: score -= 2    // сильно тяжелее

  // Восстановление
  if log.sorenessLevel >= 3: score -= 1
  if log.jointPain: score -= 2
  if log.sleepQuality == 1: score -= 1
  if log.pumpQuality == 3: score += 1  // хороший пампинг = хороший стимул

  return score  // диапазон: примерно от -6 до +4

// Шаг 2: Применить решение на основе score

ЕСЛИ score >= 3:
  → ПРОГРЕССИЯ: увеличить объём на 1-2 подхода на следующей неделе
  → Если уже на MAV → не увеличивать, держать
  → progressionCoefficient += 0.25%  (СРЦ: ускорить прогрессию)
  → Записать: «прогресс отличный, прибавляем объём и ускоряем коэффициент»

ЕСЛИ score от 1 до 2:
  → УДЕРЖАНИЕ: следовать плану мезоцикла без изменений
  → progressionCoefficient без изменений
  → Записать: «всё идёт по плану»

ЕСЛИ score от -1 до 0:
  → НАБЛЮДЕНИЕ: следовать плану, но снизить intensityPercent на 5%
  → progressionCoefficient -= 0.25%  (СРЦ: замедлить прогрессию)
  → Записать: «небольшая усталость, снижаем интенсивность»

ЕСЛИ score от -3 до -2:
  → ОТКАТ: снизить объём на 1-2 подхода, снизить intensityPercent на 10%
  → progressionCoefficient = 0%  (СРЦ: заморозить прогрессию)
  → Записать: «накопилась усталость, снижаем объём, замораживаем коэффициент»

ЕСЛИ score <= -4:
  → ЭКСТРЕННЫЙ ДЕLOAD: переключить на деload-неделю немедленно
  → progressionCoefficient = 0%, repMax не пересчитывается до конца деload
  → Записать: «нужна разгрузка, переходим к деload»

ЕСЛИ jointPain == true (в любом случае):
  → СТОП на данном упражнении: рекомендовать 7 дней отдыха для сустава
  → Предложить альтернативный вариант (резина вместо BW, меньший вес)
  → progressionCoefficient = 0%
  → Записать: «боль в суставе — берём паузу»

// Ограничения на коэффициент:
progressionCoefficient = clamp(progressionCoefficient, 0%, maxForLevel)
// maxForLevel: absolute_beginner=2.0%, beginner=1.5%, intermediate_early=1.0%,
//              intermediate_late=0.75%, advanced=0.5%
```

#### 3.5.2 Калибровка Volume Landmarks

Каждые 2 мезоцикла (или после каждого деload) система пересчитывает индивидуальные landmarks:

```
function recalibrateVolumeLandmarks(history: MesocycleLog[]):
  // Собрать данные: на каком объёме были лучшие performance scores

  weeklyData = history.flatMap(m => m.weeks.map(w => ({
    sets: w.actualSets,
    avgScore: averagePerformanceScore(w.sessions)
  })))

  // MEV = минимальный объём, при котором score > 0 стабильно
  mev = weeklyData
    .filter(d => d.avgScore > 0)
    .reduce((min, d) => Math.min(min, d.sets), Infinity)

  // MAV = объём с максимальным средним score
  mav = weeklyData
    .sort((a, b) => b.avgScore - a.avgScore)[0].sets

  // MRV = объём, после которого score стабильно падает ниже 0
  mrv = weeklyData
    .filter(d => d.avgScore < 0)
    .reduce((min, d) => Math.min(min, d.sets), Infinity)

  // Применить сглаживание (не менять больше чем на ±2 подхода за раз)
  return {
    mev: clamp(mev, oldMev - 2, oldMev + 2),
    mav: clamp(mav, oldMav - 2, oldMav + 2),
    mrv: clamp(mrv, oldMrv - 2, oldMrv + 2),
    mv: Math.max(4, mev - 2)
  }
```

### 3.6 Прогрессия в подтягиваниях (специальный алгоритм)

Для подтягиваний действует дополнительная логика поверх общей системы:

```
ЦИКЛ ПРОГРЕССИИ (длинный горизонт, несколько мезоциклов):

Фаза 1: «Набор объёма с резиной»
  Цель: довести до 4×10 с резиной при RPE 7-8
  Критерий перехода: 3 из последних 5 тренировок — 4×10 RPE ≤ 8
  (скользящее окно; один плохой день не сбрасывает счётчик)

Фаза 2: «Смешанные сессии»
  Объёмная тренировка: 4×8 с резиной
  Силовая тренировка: 3×(max BW) + 3×3 эксцентрика 5с
  Критерий перехода: max BW >= 5 повторений (2 из 3 последних тестов)

Фаза 3: «Преимущественно собственный вес»
  Объёмная тренировка: 4×5 BW
  Силовая тренировка: 3×(max BW - 1) + 2×2 эксцентрика 5с
  Критерий перехода: max BW >= 8 повторений (2 из 3 последних тестов)

Фаза 4: «Полноценная работа BW»
  Обе тренировки с собственным весом
  Применяется стандартная Double Progression
  Резина используется только при откатах/болезни

Переход между фазами автоматический по критериям (скользящее окно).
Откат на предыдущую фазу: если max BW ниже порога в 3+ из последних 5 тренировок.
```

### 3.7 Алгоритм планирования сессии (итоговый)

Это то, что пользователь видит перед тренировкой:

```
function generateSessionPlan(exercise, mesocycle, weekPlan, sessionNumber, lastSessionLog):

  1. Взять базовый план из weekPlan.sessions[sessionNumber]

  2. Применить авторегуляцию (если есть lastSessionLog):
     adjustment = getVolumeAdjustment(lastSessionLog)  // из раздела 3.5.1
     plan.sets += adjustment.setsChange
     plan.targetRPE += adjustment.rpeChange

  3. Для подтягиваний — определить режим:
     pullupPhase = getCurrentPullupPhase(exercise, history)
     plan.loadConfig = getPullupLoadConfig(pullupPhase, sessionNumber)

  4. Для гантелей — определить вес:
     plan.loadConfig = getDumbbellWeight(exercise, lastSessionLog)

  5. Применить ограничения безопасности:
     plan.sets = clamp(plan.sets, exercise.volumeLandmarks.mv, exercise.volumeLandmarks.mrv)
     plan.targetRPE = clamp(plan.targetRPE, 6, 9)
     plan.reps = max(1, plan.reps)

  6. Вернуть план:
     return {
       exercise: exercise.name,
       sets: plan.sets,
       reps: plan.reps,
       targetRPE: plan.targetRPE,
       loadDescription: formatLoadConfig(plan.loadConfig),
       notes: generateNotes(adjustment, pullupPhase)
     }
```

## 6. Пользовательские сценарии (User Flows)

### 6.1 Первый запуск

```
1. Ввод профиля (вес, рост, возраст, стаж, частота тренировок)
2. Добавление первого упражнения
   → Выбор типа (BW / гантели)
   → Для BW: тест — сколько можешь подтянуться? с резиной?
   → Для гантелей: отметить доступные веса
3. Система генерирует первый мезоцикл
4. Показ плана первой тренировки с объяснением
```

### 6.2 Обычная тренировка

```
1. Открыть приложение → видишь план на сегодня
2. Нажать «Начать тренировку»
3. Для каждого подхода:
   → Увидеть цель (повторения, RPE)
   → Выполнить подход
   → Ввести фактические повторения + RPE
   → Запускается таймер отдыха
4. После всех подходов → заполнить обратную связь
5. Увидеть Performance Score и решение системы
6. Увидеть план следующей тренировки (предварительно)
```

### 6.3 Конец мезоцикла

```
1. После последней рабочей недели → система предлагает деload
2. Деload неделя: облегчённые тренировки
3. После деload → система рекалибрует Volume Landmarks
4. Генерация нового мезоцикла
5. Показ сравнения: «Мезоцикл 1 vs Мезоцикл 2»
   → Изменение максимумов
   → Изменение объёмных ориентиров
```

### 6.4 Пропуск тренировки

```
1. Если тренировка пропущена:
   → На следующую тренировку: план не меняется
   → Если пропущено 2+ тренировки подряд:
     → Снизить план на 1 подход и RPE на 0.5
     → Показать: «Давно не тренировались, начинаем мягче»
   → Если пропущено 2+ недели:
     → Перезапустить мезоцикл с MEV
```

### 6.5 Планирование отпуска / каникул

Пользователь может заранее отметить в календаре период, когда тренировок не будет.

```
Модель данных:

interface VacationPeriod {
  id: string;
  startDate: Date;
  endDate: Date;
  label?: string;           // «Отпуск», «Командировка» и т.д.
  partialTraining: boolean;  // true = будет доступ к тренировкам, но ограниченно
}

Алгоритм адаптации мезоцикла к отпуску:

ВХОД: activeMesocycle, vacationPeriod

1. Вычислить: сколько рабочих недель до отпуска (weeksBeforeVacation)
   и сколько после (weeksAfterVacation, если мезоцикл не закончен)

2. ЕСЛИ vacationDuration <= 7 дней (1 неделя):
   → Заменить эту неделю на деload
   → Если деload уже был на прошлой неделе — просто пропустить (пауза)
   → После отпуска: продолжить мезоцикл с того же уровня объёма
   → Показать: «Отпуск совпадает с разгрузочной неделей — отличное время!»

3. ЕСЛИ vacationDuration = 8-14 дней (2 недели):
   → Неделя 1 отпуска: деload (если partialTraining = true) или пауза
   → Неделя 2: пауза
   → После отпуска: вернуться на 1 неделю назад в мезоцикле
     (т.е. повторить последнюю рабочую неделю перед отпуском)
   → RPE первой тренировки после отпуска: -1 от плана
   → Показать: «Возвращаемся мягко — повторяем прошлую рабочую неделю»

4. ЕСЛИ vacationDuration > 14 дней:
   → Завершить текущий мезоцикл досрочно
   → После отпуска: новый мезоцикл с MEV
   → Рамп-ап тренировка: 75% от MEV, RPE 5-6
   → Показать: «Длинный перерыв — начинаем новый цикл с нуля»

5. Для bodyweight упражнений (подтягивания):
   → Если отпуск > 14 дней: пересдать тест максимумов (BW и резина)
   → Если отпуск > 21 дней: возможен откат progressionPhase на 1 уровень

6. ЕСЛИ partialTraining = true и vacationDuration <= 14 дней:
   → Предложить мини-план: 1 тренировка в неделю на MV (поддержание)
   → Только ключевые упражнения, RPE 6-7
   → Показать: «План поддержания на время поездки»

Автоматическая подстройка календаря:
  - При добавлении отпуска система пересчитывает даты всех оставшихся недель
  - Деload сдвигается или совмещается с отпуском
  - Новый мезоцикл после отпуска стартует автоматически

UI:
  - В настройках: раздел «Календарь» с возможностью отметить даты
  - Визуально: отпуск отображается серой полосой на графике прогресса
  - За 1 неделю до отпуска: уведомление «Скоро отпуск. Текущую неделю делаем
    на полную, на следующей — разгрузка / пауза»
```

## 8. Формулы и константы (справочник)

### 8.1 Performance Score

```
score = repCompletion + rpeAccuracy + recoveryFactors

repCompletion:
  ratio = actualReps / plannedReps
  if ratio >= 1.05: +2
  if ratio >= 0.95: +1
  if ratio >= 0.90: 0
  if ratio >= 0.80: -1
  if ratio < 0.80: -2

rpeAccuracy:
  deviation = actualRPE - targetRPE
  if deviation <= -1.0: +1
  if deviation <= 0.5: 0
  if deviation <= 1.0: -1
  if deviation > 1.0: -2

recoveryFactors:
  soreness >= 3: -1
  jointPain: -2
  sleepQuality == 1: -1
  pumpQuality == 3: +1
  pumpQuality == 1: -1

Итого: от -9 до +4
```

### 8.2 Маппинг Performance Score → Решение

| Score | Решение | Подходов | Интенсивность | Коэфф. прогрессии |
|-------|---------|----------|---------------|-------------------|
| >= 3 | ПРОГРЕССИЯ | +1..+2 | без изменений | +0.25% |
| 1..2 | УДЕРЖАНИЕ | 0 | без изменений | без изменений |
| -1..0 | НАБЛЮДЕНИЕ | 0 | -5% | -0.25% |
| -3..-2 | ОТКАТ | -1..-2 | -10% | → 0% |
| <= -4 | ДЕLOAD | → MV | → light (40-55%) | → 0% |
| jointPain | СТОП | 0 (пауза) | — | → 0% |

### 8.7 Double Progression для гантелей

```
Если maxReps >= repRange.max:
  weight = nextAvailableWeight
  Если jumpPercent > 20%:
    repRange.min -= 2  (расширить диапазон вниз)
  reps = repRange.min

Если maxReps < repRange.min:
  weight = previousAvailableWeight
  reps = repRange.min
```

### 8.8 Деload параметры

```
sets = volumeLandmarks.mv
reps = round(normalWeek1Reps * 0.5)
targetRPE = 5-6

Выбор нагрузки на деload:
  Для bodyweight упражнений:
    Если progressionPhase <= 2 → mode = 'band_assisted' (всегда с резиной)
    Если progressionPhase == 3 → mode = 'band_assisted' (откат к резине на деload)
    Если progressionPhase == 4 → mode = 'bodyweight' (но сниженные повторения)
  Для гантелей:
    weight = нагрузка Недели 1 (без прогрессии)
```

### 8.9 Коррекция Volume Landmarks по относительной силе

Дефолтные landmarks (раздел 8.3) корректируются для тяжёлых упражнений:

```
function adjustLandmarks(defaults, exercise, userProfile):
  // Коэффициент сложности: отношение BW к силе
  if exercise.type == 'bodyweight':
    if userProfile.bodyweight > 100:
      difficultyFactor = 0.75  // снизить все landmarks на 25%
    elif userProfile.bodyweight > 85:
      difficultyFactor = 0.85
    else:
      difficultyFactor = 1.0

    // Дополнительная коррекция по текущему уровню
    if exercise.currentMaxReps <= 3:
      difficultyFactor *= 0.8  // ещё -20% если макс <= 3 повтора

    return {
      mv: max(3, round(defaults.mv * difficultyFactor)),
      mev: max(4, round(defaults.mev * difficultyFactor)),
      mav: max(6, round(defaults.mav * difficultyFactor)),
      mrv: max(10, round(defaults.mrv * difficultyFactor))
    }

  // Для гантелей landmarks не корректируются
  return defaults

ПРИМЕР для атлета 117кг, maxBW=3, стаж 2-5 лет:
  difficultyFactor = 0.75 * 0.8 = 0.60
  mv  = max(3, round(6 * 0.6)) = max(3, 4) = 4
  mev = max(4, round(10 * 0.6)) = max(4, 6) = 6
  mav = max(6, round(16 * 0.6)) = max(6, 10) = 10
  mrv = max(10, round(22 * 0.6)) = max(10, 13) = 13
```

### 8.10 Протокол восстановления после болезни / длительного перерыва

```
Если перерыв 1-2 недели:
  → Перезапустить мезоцикл: новый мезоцикл с MEV, Неделя 1
  → Первая тренировка: RPE 6 (на 1 ниже обычного старта)
  → Вторая тренировка: RPE 7 (штатный уровень)

Если перерыв 3-4 недели:
  → Новый мезоцикл с MEV
  → Рамп-ап неделя: 75% от MEV, RPE 5-6 (не считается рабочей неделей)
  → Затем стандартный мезоцикл

Если перерыв 5+ недель:
  → Откат progressionPhase на 1 уровень (для bodyweight)
  → Пересдать тест максимумов (BW и резина)
  → Новый мезоцикл по обновлённым данным

Если хроническая боль в суставе (> 2 недель):
  → Заменить упражнение на альтернативное (предложить пользователю)
  → Или: полностью перейти на band_assisted / эксцентрику
  → Не возвращаться к проблемному режиму без подтверждения пользователя
```

### 8.11 Минимальные ограничения Double Progression

```
Для гантелей:
  Минимум повторений после расширения диапазона: 3
  repRange.min = max(3, repRange.min - expansion)

  Если nextAvailableWeight не существует (максимальный вес):
  → Переключиться на увеличение подходов (3×max → 4×max → 5×max)
  → Предупредить: «Достигнут максимальный вес гантели, добавляем подходы»

  Если availableWeights пустой или только 1 вес:
  → Работать только с прогрессией повторений и подходов
```

### 8.12 Безопасные условия для калибровки landmarks

```
function recalibrateVolumeLandmarks(history):
  // Минимум данных для калибровки
  if totalWeeksWithData < 6:
    return currentLandmarks  // не менять, недостаточно данных

  // Защита от невалидных значений
  if calculatedMEV > calculatedMAV:
    calculatedMEV = min(currentMEV, calculatedMEV)

  if calculatedMAV > calculatedMRV:
    calculatedMAV = currentMAV  // не менять MAV

  // Не менять больше чем на ±2 подхода за одну калибровку
  // Требовать минимум 3 точки данных на каждый объёмный уровень
```

## 10. Глоссарий

| Термин | Определение |
|--------|-------------|
| **Мезоцикл** | Тренировочный блок из 4-6 недель с нарастающим объёмом и завершающим деload |
| **Деload** | Разгрузочная неделя со сниженным объёмом и интенсивностью для восстановления |
| **RPE** | Rate of Perceived Exertion — субъективная оценка тяжести подхода (6-10) |
| **RIR** | Reps in Reserve — сколько повторений осталось «в запасе» |
| **MEV** | Minimum Effective Volume — минимальный объём для стимуляции роста |
| **MAV** | Maximum Adaptive Volume — зона оптимального объёма |
| **MRV** | Maximum Recoverable Volume — потолок восстанавливаемого объёма |
| **MV** | Maintenance Volume — объём для удержания текущих результатов |
| **Hard set** | Подход, выполненный с RPE 7+ (в пределах 3 повторений до отказа) |
| **Double Progression** | Метод: сначала увеличиваем повторения до верха диапазона, затем увеличиваем вес |
| **Эксцентрика** | Фаза опускания (негативная) — медленное контролируемое опускание |
| **Performance Score** | Вычисляемый показатель качества тренировки для принятия решений авторегуляции |

## Приложение A: Научные источники

Алгоритмическая модель приложения основана на гибриде российской и западной школ:

**Российская школа (основа периодизации и прогрессии):**
1. **СРЦ — Саморасчитывающиеся Циклы** (Дмитрий Головинский) — процентный подход от repMax, еженедельный пересчёт с коэффициентом прогрессии, развязка объёма и интенсивности, волновое микроциклирование (тяжёлая/лёгкая сессия)
2. **Block Periodization** (Vladimir Issurin) — последовательная концентрация тренировочных стимулов
3. **Tudor Bompa** — классическая и волнообразная периодизация, Periodization Training for Sports

**Западная школа (авторегуляция и volume landmarks):**
4. **Renaissance Periodization** (Mike Israetel) — система Volume Landmarks (MEV/MAV/MRV), калибровка объёма по данным атлета
5. **3D Muscle Journey** (Eric Helms) — The Muscle and Strength Training Pyramid, RPE/RIR как корректор
6. **Reactive Training Systems** (Mike Tuchscherer) — RPE-based autoregulation для силовых тренировок
7. **Stronger By Science** (Greg Nuckols) — индивидуальная вариативность объёма, систематический подбор
