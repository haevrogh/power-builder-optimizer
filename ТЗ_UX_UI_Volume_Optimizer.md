# Техническое задание: UX/UI — Volume Optimizer PWA

## Дизайн-система, компоненты и спецификации интерфейса

**Версия:** 1.0
**Дата:** 28.03.2026
**Стиль:** Светлая тема, минималистичный, вдохновлён Apple Health/Fitness + Material Design 3

---

## 1. Дизайн-философия

### 1.1 Принципы

**Не нейрослоп.** Никаких градиентных кнопок, теней от всего подряд, случайных иконок и перегруженных экранов. Каждый элемент на экране существует потому что решает задачу.

Три ключевых принципа:

**Ясность.** В зале с телефоном в мокрых руках ты должен за 1-2 секунды понять: сколько делать, с чем, какой RPE. Крупные числа, минимум текста, контрастные зоны.

**Тишина.** Между элементами — воздух. Белый фон не значит пустоту, он значит фокус. Информация появляется когда нужна. Вторичные данные — в bottom sheet, не на главном экране.

**Ритм.** Приложение работает в ритме тренировки: подход → ввод → отдых → подход. UI следует этому циклу, а не навязывает свой.

### 1.2 Визуальный референс

Apple Health / Fitness: чистая типографика, кольцевые прогресс-индикаторы, карточки с данными, спокойная палитра с точечными акцентами цвета.

## 2. Типографика

### 2.1 Шрифтовой стек

Все шрифты — бесплатные, на Google Fonts, с отличной кириллицей.

| Роль | Шрифт | Вес | Почему |
|------|-------|-----|--------|
| **Заголовки** | **Onest** | Bold 700 | Геометрический гротеск, гибрид гуманистического и геометрического. 8.5/10 кириллица. Variable font — один файл на все веса |
| **Основной текст / UI** | **Golos Text** | Regular 400, Medium 500 | Разработан Paratype (Россия) специально для кириллицы. 9/10 качество. Вариант «Golos UI» оптимизирован для интерфейсов |
| **Числа / данные** | **IBM Plex Mono** | Regular 400 | Моноширинный, отличная кириллица. Числа одинаковой ширины — колонки данных выравниваются идеально |

### 2.3 Правила для кириллицы

- **Line-height**: для кириллицы 1.5-1.6 (на 0.1 больше чем для латиницы) — буквы визуально выше
- **Letter-spacing**: +0.02-0.04em для body-текста — кириллица шире латиницы на 15-20%
- **Вес**: для body-текста предпочитать Medium (500) при размерах <14px — тонкие начертания плохо читаются
- **Никогда**: не использовать Inter для кириллицы (качество 4/10, несмотря на популярность)
- **Подключение**: variable font через Google Fonts, subsetting до кириллицы + латиницы (экономия 40-60% размера файла)

```css
/* Подключение шрифтов */
@import url('https://fonts.googleapis.com/css2?family=Golos+Text:wght@400;500;700&family=Onest:wght@400;500;700&family=IBM+Plex+Mono:wght@400;500&subset=cyrillic&display=swap');
```

- Цветные элементы — точечные: кольца прогресса, чипы состояния, акцентные кнопки
- Графики — единственное место где допустимо 3+ цветов рядом
- Текст — только `on-surface` и `on-surface-variant`. Цветной текст — исключительно для ссылок и статусов

## 4. Сетка и отступы

### 4.1 Базовая единица

**8px** — все размеры и отступы кратны 8. Исключение: 4px для микроотступов (между иконкой и текстом).

### 4.2 Токены отступов

| Токен | Значение | Где |
|-------|----------|-----|
| `space-xs` | 4px | Между иконкой и текстом, внутренние micro-gaps |
| `space-sm` | 8px | Между элементами внутри компонента |
| `space-md` | 16px | Padding карточек, отступы секций |
| `space-lg` | 24px | Между секциями экрана |
| `space-xl` | 32px | Между крупными блоками |
| `space-2xl` | 48px | Верхний отступ экрана, нижний safe area |

### 4.3 Ширина контента

- **Максимальная ширина**: 428px (iPhone 15 Pro Max logical width)
- **Боковые отступы экрана**: 16px с каждой стороны
- **Контентная зона**: 396px максимум
- **На планшете**: контент центрируется, максимум 600px

### 4.4 Safe areas

- **Верхний отступ**: динамический (iOS safe area inset)
- **Нижний отступ**: 34px (home indicator) + 56px (bottom nav) = 90px
- **Контент не должен** уходить под bottom navigation

## 5. Компоненты

### 5.1 Карточка данных (Data Card)

Основной строительный блок — вдохновлён Apple Health:

```
┌─────────────────────────────┐  ← border-radius: 16px
│                             │  ← background: surface-dim (#F5F5F7)
│  Подтягивания          ●→   │  ← headline-large (Onest 28/700)
│                             │     ●→ = иконка-стрелка для перехода
│  ┌──────┐  ┌──────┐        │
│  │  3   │  │  10  │        │  ← data-large (IBM Plex Mono 32/400)
│  │макс  │  │резина│        │  ← label-medium (#8E8E93)
│  │ BW   │  │      │        │
│  └──────┘  └──────┘        │
│                             │
│  Фаза 2 · Коэфф 0.75%     │  ← body-medium (#8E8E93)
│  ████████░░░░░ до Фазы 3   │  ← progress bar
│                             │
│  padding: 16px              │
│  gap между элементами: 12px │
└─────────────────────────────┘
```

**Спецификация:**
- `background`: surface-dim
- `border-radius`: 16px
- `padding`: 16px
- `box-shadow`: none (без теней — чистый минимализм)
- `border`: 1px solid outline-variant (тонкая граница, opacity 0.5)
- `gap`: 12px между дочерними элементами

### 5.2 Кольцо прогресса (Progress Ring)

По мотивам Apple Fitness Activity Rings:

```
        ╭───────╮
      ╱   75%     ╲      ← stroke-width: 12px
    │    ┌───┐      │    ← внутри: текущее значение
    │    │ 6 │      │       data-large (IBM Plex Mono)
    │    │/8 │      │       "/8" — label-medium, variant color
    │    └───┘      │
      ╲           ╱
        ╰───────╯
```

**Спецификация:**
- Диаметр: 120px (крупное кольцо) / 80px (среднее) / 48px (мини)
- `stroke-width`: 12px (крупное) / 8px (среднее) / 6px (мини)
- `stroke-linecap`: round
- Фоновый трек: outline-variant (#E5E5EA)
- Прогресс: цвет зависит от контекста (progress/intensity/primary)
- Анимация заполнения: 600ms, ease-out

### 5.3 Ввод повторений (Rep Input)

**Критичный компонент.** В зале, в перчатках, с мокрыми руками — ввод должен быть за 1-2 тапа.

#### Вариант: Tap-to-Set с крупными кнопками

```
┌─────────────────────────────────┐
│                                 │
│    ┌───┐               ┌───┐   │
│    │ − │      8        │ + │   │  ← кнопки 56×56px
│    └───┘               └───┘   │  ← число: display-medium (36px)
│                                 │
│    Цель: 8 · RPE 7-8           │  ← label-medium, variant color
│                                 │
└─────────────────────────────────┘
```

**Спецификация:**
- Кнопки ±: **56×56px** (больше чем MD3 минимум 48px — для перчаток)
- `border-radius` кнопок: 16px
- `background` кнопок: surface-container
- `color` кнопок: on-surface
- Число в центре: display-medium (Onest 36px Bold)
- Tap на число — открывает numpad для прямого ввода (fallback)
- Haptic feedback: light impact на каждый тап ±
- **Long press** на ± : ускоренный инкремент (каждые 150ms)

### 5.4 Селектор RPE

Горизонтальный ряд чипов (Material 3 Filter Chips):

```
┌─────────────────────────────────────┐
│  RPE этого подхода:                 │
│                                     │
│  ┌──┐ ┌──┐ ┌────┐ ┌──┐ ┌──┐       │
│  │ 6│ │ 7│ │ ●8 │ │ 9│ │10│       │
│  └──┘ └──┘ └────┘ └──┘ └──┘       │
│                                     │
│  ░░░░░████████░░░░░░░░░░░░░        │
│  легко  ← целевая зона →  тяжело   │
│                                     │
└─────────────────────────────────────┘
```

**Спецификация:**
- Каждый чип: минимум 48×40px
- Выбранный чип: `background` = primary, `color` = white
- Невыбранные: `background` = surface-container, `color` = on-surface
- `border-radius`: 12px
- `gap`: 8px
- Под чипами: тонкая визуальная шкала с подсветкой целевой зоны (primary-container)
- Haptic: medium impact при выборе

### 5.5 Селектор веса гантели

Scroll-wheel с крупными значениями для дискретных весов:

```
┌─────────────────────────────┐
│  Вес:                       │
│       ┊                     │
│      22  ← полупрозрачный   │
│      25  ← ВЫБРАНО (крупно) │  ← data-large, primary color
│      27  ← полупрозрачный   │
│       ┊                     │
│                             │
└─────────────────────────────┘
```

**Спецификация:**
- Вертикальный scroll с snap-to-item
- Видимы 3 значения: предыдущий, текущий, следующий
- Текущий: data-large (32px), `color` = primary
- Соседние: data-medium (20px), `color` = on-surface-variant, `opacity` = 0.5
- Высота контейнера: 120px
- Snap behavior: `scroll-snap-type: y mandatory`
- Haptic: light tick на каждом значении при прокрутке

## 8. Анимации и микровзаимодействия

### 8.1 Токены анимации

| Токен | Длительность | Easing | Где |
|-------|-------------|--------|-----|
| `motion-quick` | 150ms | ease-out | Состояния кнопок, тогглы |
| `motion-standard` | 300ms | ease-in-out | Переходы между экранами, bottom sheet |
| `motion-emphasis` | 500ms | ease-out | Появление кольца прогресса, score |
| `motion-long` | 600ms | ease-out | Заполнение progress ring |

### 8.2 Ключевые анимации

| Момент | Анимация |
|--------|----------|
| Запись подхода | Кольцо прогресса заполняется на долю (600ms). Записанный подход появляется снизу (slide-up 300ms) |
| Переход к таймеру | Bottom sheet поднимается (300ms). Кольцо таймера начинает убывать |
| Завершение таймера | 3 haptic-пульса. Кольцо мигает (opacity pulse 2 раза) |
| Performance Score | Число «вкатывается» (count-up анимация от 0 до значения, 500ms) |
| Переход фазы | Конфетти / пульсирующее кольцо (1 раз, не навязчиво) |
| Переключение табов | Cross-fade 150ms (без slide — чище) |
| Появление карточек | Stagger: каждая следующая карточка появляется с задержкой 50ms |

### 8.3 Haptic feedback

| Действие | Тип вибрации |
|----------|-------------|
| Тап кнопки ± | Light impact |
| Выбор RPE чипа | Medium impact |
| Прокрутка веса (каждый snap) | Light tick |
| Запись подхода | Success notification |
| Завершение таймера | 3× short pulse |
| Завершение тренировки | Success notification |
| Экстренный деload | Warning notification |

## 9. Адаптивность и gym-friendly UX

### 9.1 Touch targets

- **Минимум**: 48×48px для всех интерактивных элементов
- **Предпочтительно**: 56×56px для primary-действий (ввод повторений, RPE)
- **Зазор между targets**: минимум 8px
- Причина: в зале руки мокрые, в перчатках, телефон может быть на скамье — точность тапа снижена

### 9.2 Однорукий режим

- Все primary-действия — в нижней 2/3 экрана (зона большого пальца)
- Bottom navigation — нижний край
- Primary CTA — всегда над bottom nav
- Информация, которую не нужно нажимать (заголовки, статистика) — вверху

### 9.3 Быстрый ввод

- Цель: запись одного подхода за **3 тапа максимум** (повторения ±, RPE, записать)
- Если повторения = целевым — предзаполнены, RPE = предыдущему — предзаполнен
- В идеале: **1 тап** (нажать «Записать» если всё совпало с планом)

### 9.4 Ориентация

- **Только portrait**. В зале никто не переворачивает телефон
- Графики на экране прогресса: горизонтальный скролл для длинных периодов

### 9.5 Контрастность (WCAG AA)

Все текстовые пары проверены на соответствие WCAG AA (минимум 4.5:1 для текста, 3:1 для крупного):

| Текст | Фон | Контраст | Статус |
|-------|-----|----------|--------|
| on-surface (#1C1C1E) | surface (#FFFFFF) | 16.7:1 | AA ✓ |
| on-surface (#1C1C1E) | surface-dim (#F5F5F7) | 14.8:1 | AA ✓ |
| on-surface-variant (#8E8E93) | surface (#FFFFFF) | 3.5:1 | AA крупный текст ✓ |
| on-surface-variant (#8E8E93) | surface-dim (#F5F5F7) | 3.1:1 | AA крупный текст ✓ |
| primary (#007AFF) | surface (#FFFFFF) | 4.6:1 | AA ✓ |
| primary (#007AFF) | primary-container (#E3F2FF) | 4.1:1 | AA крупный текст ✓ |
| white (#FFFFFF) | primary (#007AFF) | 4.6:1 | AA ✓ |
| danger (#FF3B30) | surface (#FFFFFF) | 3.9:1 | AA крупный текст ✓ |

**Правило:** `on-surface-variant` (#8E8E93) используется ТОЛЬКО для текста ≥14px Medium (500) или ≥16px Regular. Для мелкого текста — только `on-surface`.

### 9.6 Отмена действия (Undo)

В зале легко промахнуться. Критичные действия должны быть отменяемы:

| Действие | Undo |
|----------|------|
| Записать подход | Snackbar внизу: «Подход записан · Отменить» (5 сек, потом исчезает) |
| Завершить тренировку | Экран подтверждения: «Завершить? Все подходы записаны» |
| Удалить упражнение | Диалог подтверждения + 30 дней в «корзине» |
| Экстренный деload | Информационный экран с объяснением + кнопка «Вернуть план» |

Snackbar: высота 48px, padding 16px, `background` = on-surface (#1C1C1E), `color` = white, `border-radius` = 12px. Появляется над bottom nav.

### 9.7 Disabled и error состояния

| Элемент | Disabled | Error |
|---------|----------|-------|
| Кнопка | `opacity`: 0.38, не реагирует на тап | Не применимо |
| Rep input | Числа серые (on-surface-variant), кнопки ± не реагируют | Красная обводка (danger) при вводе <1 или >50 |
| RPE чипы | Все серые, не кликабельны | Красная подсветка целевого если факт. RPE > целевого на 2+ |
| Scroll wheel | Серые значения, не скроллится | Не применимо |
| Карточка сессии | `opacity`: 0.5, надпись «Пауза» или «Деload» | Не применимо |

## 10. PWA-специфика

### 10.1 Splash screen

- Фон: surface (#FFFFFF)
- По центру: логотип приложения (иконка) + название (Onest Bold 28px)
- Без анимации загрузки — мгновенный переход к контенту

### 10.2 Install prompt

Показывать после 3-й сессии использования:
- Bottom sheet: «Добавить на главный экран для быстрого доступа»
- Две кнопки: «Установить» (primary), «Позже» (text)

### 10.3 Offline-first

- Все данные локально (IndexedDB)
- Никаких спиннеров загрузки — данные всегда доступны
- При экспорте (JSON backup): единственный момент когда нужна сеть — email / share

### 10.4 Иконка приложения

- Стиль: плоская, минималистичная
- Содержание: стилизованная гантель или кольцо прогресса
- Цвета: primary (#007AFF) на белом фоне
- Формат: 512×512 PNG + maskable variant для Android
